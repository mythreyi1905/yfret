YFret Programming Test

Creating a webservice that can handle the following requests and deploying it on a kubernetes cluster

1. Show how many records that do not have "cellular" contact ?

The total number of records falling under this category along with the records itself can be accessed, if you hit the URL in the address bar available at the time of accessing by appending "/query1" to it(URL/query1)


For Example : https://285b19875611.ngrok.io/query1


2.Use "day" and "month" column values and find records which are after 15th October

The records falling under this category can be accessed, if you hit the URL in the address bar available at the time of accessing by appending "/query2" to it(URL/query2)


For Example : https://285b19875611.ngrok.io/query2


3.How many records having a specific marital status and between certain age group? Example : start_age : 20, end_age: 50, marital_status = 'married'

This is slightly tricky XD . Instead of adding one parameter to the URL, we will have to add 4 parameters.They are "/query3", "/start_age", "/end_age" and "/marital_status"(URL/query3/start_age/end_age/marital_status)


For Example: https://285b19875611.ngrok.io/query3/15/49/divorced


15-start_age : Integer value, the minimum age you want to specify

49 - end_age : Integer value, the maximum age that you want to specify

divorced - marital_status : Holds values "single" or "divorced" or "married"


Note : You willl encounter a 404 error if you miss out on these parameters. (URL/query3 is not valid)

The scripting of the webservice has been done using Python Flask + MongoDB and has been deployed as a docker container orchestrated by kubernetes(minikube)
